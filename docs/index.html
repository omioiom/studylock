이게 뭔가요?

StudyLock은 허용한 앱만 열리게 하는 자기통제 잠금 앱입니다.

허용 앱만 사용

내가 고른 공부 앱만 열리고 나머지는 차단.

시간표 · 스크린타임

하루 계획, 앱별/시간대 제한, 일정 알림.

기기관리자 잠금

삭제·안전모드·설정 우회까지 시스템 레벨 차단.

소스 공개

코드 전부 공개(CC BY-NC-ND · 개인·비상업). 서버 없음, 데이터는 기기에만.

준비물

기기관리자 등록은 아래 조건이 필요해요. (한 번만 하면 끝)

PC + 크롬/엣지

이 도구는 데스크톱 Chrome·Edge에서만 동작(WebUSB). 폰 브라우저는 불가.

USB 케이블

폰을 PC에 USB로 연결.

USB 디버깅 ON

개발자 옵션에서 켜기 (아래 안내).

계정 0개

기기관리자 등록은 구글·삼성 계정이 없어야 성공. 있으면 제거 또는 초기화.

USB 디버깅 켜는 법

삼성 원UI 기준. 기종마다 이름이 조금 다를 수 있어요.

  1. 설정 → 보안 및 개인정보 보호 → 보안 위험 자동 차단을 먼저 해제
  2. 설정 → 휴대전화 정보 → 소프트웨어 정보
  3. 빌드번호를 7번 연속 탭 → "개발자가 되었습니다" 문구확인
  4. 설정 (스크롤하여 맨 밑으로) → 개발자 옵션 진입
  5. USB 디버깅 켜기

'보안 위험 자동 차단'을 왜 끄나요? — 삼성의 이 기능은 출처와 상관없이 USB(케이블)로 하는 앱 설치를 통째로 막아요. 스터디락이 위험한 앱이라서가 아니라, 이 도구가 쓰는 USB 설치 방식 자체가 차단 대상이라 잠깐 꺼두는 거예요. 설치가 끝난 뒤에는 다시 켜도 스터디락은 정상 동작해요.

설치하기

아래 3개를 확인하고, 폰을 연결한 뒤 원클릭 설치하세요.

USB 디버깅 켜는 법 (삼성 갤럭시)
  1. 설정 → 보안 및 개인정보 보호 → 보안 위험 자동 차단 먼저 해제
  2. 설정 → 휴대전화 정보 → 소프트웨어 정보
  3. 빌드 번호를 7번 연속 탭 → "개발자 모드 켜짐"
  4. 설정으로 나와 개발자 옵션 진입
  5. USB 디버깅 켜기

'보안 위험 자동 차단'은 USB(케이블)로 하는 앱 설치를 전부 막는 삼성 기본 기능이에요. 스터디락이 이상한 앱이라서가 아니라 설치 방식 때문이니 안심하세요. 설치 후 다시 켜도 돼요.

계정 0개 만드는 법 (삼성 갤럭시)
  1. 설정 열기
  2. 계정 및 백업 → 계정 관리
  3. 삭제할 계정 클릭 → 계정 삭제
  4. 계정 관리에 계정이 하나도 안 남을 때까지 반복

만약 계정을 0개로 만들었는데 문제가 발생한 경우 문제발생시 생기는 AI 빠른 해결 버튼을 누르면 빠르고 쉽게 해결 가능해요



      
    


    
이 브라우저는 WebUSB를 지원하지 않아요. 데스크톱 Chrome 또는 Edge로 열어주세요. (아래 수동 설치도 가능)

  


  

    
수동 설치 (adb)

    
위 원클릭이 안 되면, PC에 adb 깔고 아래 두 줄이면 끝이에요.

    
adb install -r StudyLock.apk

    
adb shell dpm set-device-owner com.studylock/.AdminReceiver

    
두 번째 명령이 실패하면 → 폰에 계정이 남아있는 것. 전부 제거(또는 초기화) 후 다시.

    
StudyLock.apk 직접 받기
